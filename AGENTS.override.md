# Project-Specific Overrides

User-owned; survives `rulebook update`; wins on conflict.

- Execute requests end-to-end in one turn — no mid-task "should I proceed?"
  check-ins. Ask only for genuine ambiguity that changes the outcome, or for
  destructive operations.
- Batch related work into few commits; report once at the end.
- Subagents and parallel dispatch are encouraged whenever work is parallel or
  context-heavy.
