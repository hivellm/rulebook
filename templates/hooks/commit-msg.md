# Commit Message Hook

Runs after the commit message is written (`.git/hooks/commit-msg`, receives the message file path as `$1`) and rejects messages that do not follow Conventional Commits. Consistent messages enable automated changelogs and semantic versioning.

## Required format

```
<type>(<scope>): <subject>

<body>

<footer>
```

- **type** (required): `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`
- **subject** (required): imperative mood, lowercase start, no trailing period, ≤ 50 chars (hard cap 72)
- **scope** (optional): codebase area, e.g. `feat(auth): ...`
- **body** (optional): the "why", wrapped at 72 chars, blank line after subject
- **footer** (optional): `BREAKING CHANGE: ...` (or `feat!: ...`), `Closes #123`, `Co-authored-by: ...`

Examples: `feat(auth): add JWT token validation` · `fix: prevent race condition in task queue`

## Minimal shell hook

```bash
#!/bin/sh
MSG=$(head -1 "$1")
PATTERN='^(feat|fix|docs|style|refactor|perf|test|build|ci|chore|revert)(\([a-z0-9-]+\))?!?: .{1,72}$'
if ! echo "$MSG" | grep -qE "$PATTERN"; then
  echo "Invalid commit message: '$MSG'"
  echo "Expected: <type>(<scope>): <subject>  e.g. feat(api): add pagination"
  exit 1
fi
# Allow merge/revert autogenerated messages
echo "$MSG" | grep -qE '^(Merge|Revert)' && exit 0
exit 0
```

Install with `cp scripts/commit-msg.sh .git/hooks/commit-msg && chmod +x .git/hooks/commit-msg`, or use commitlint via husky: `npx husky set .husky/commit-msg 'npx --no -- commitlint --edit "$1"'`.

## How to fix failures

- **Rejected type**: pick the closest allowed type; `chore` covers non-src/test changes.
- **Subject too long**: move detail to the body; the subject states what, the body states why.
- **Merge/revert commits blocked**: whitelist `^Merge` / `^Revert` prefixes in the hook.
- **Bypass** (emergencies only): `git commit --no-verify` — then fix the history before pushing.
