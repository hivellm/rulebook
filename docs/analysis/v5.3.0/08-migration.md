# 08 — Migration Path for Existing v5.2.x Projects

1. **`rulebook update`** detects a v5.2-style CLAUDE.md (no `@import`, no `RULEBOOK:START v5.3.0` sentinel).
2. Prompt: "CLAUDE.md will be refactored to use @imports (Anthropic official format). A backup will be saved to `.rulebook/backup/<ts>/CLAUDE.md`. Continue? [Y/n]"
3. Extract project-specific content (anything not matching known v5.2 templates) → append to `AGENTS.override.md`.
4. Write new thin CLAUDE.md from `CLAUDE_MD_v2.md` template.
5. Generate missing `.claude/rules/*.md` for detected languages.
6. Bootstrap `.rulebook/STATE.md` from current task state, `CLAUDE.local.md` stub, `.gitignore` entries.
7. Show a diff summary and link to backup.
8. `rulebook doctor` runs automatically post-update.

## No-break guarantees

- AGENTS.md content is untouched (only *referenced* by CLAUDE.md now).
- `.rulebook/decisions/`, `knowledge/`, `learnings/`, `memory/` are never modified by migration.
- `.claude/agents/`, `.claude/hooks/`, `.claude/statusline.sh` are never touched.
- Existing custom `.claude/rules/*.md` without the `# Generated by rulebook` sentinel are preserved.
- Active tasks in `.rulebook/tasks/*/tasks.md` keep their tail prompt **opt-in** (see [06-task-tail.md](./06-task-tail.md)) — never silently rewritten.
