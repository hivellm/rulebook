# Rulebook v5 Architecture

## 3-Layer Multi-Tool Architecture

```
Layer 1: Universal Directives (any LLM reads)
├── AGENTS.md                          — Index + core rules
├── .rulebook/specs/TIER1_PROHIBITIONS.md  — 7 absolute prohibitions
├── .rulebook/specs/TOKEN_OPTIMIZATION.md  — Model tier assignment
├── .rulebook/specs/GIT.md             — Allow-list/forbidden-list
├── .rulebook/specs/QUALITY_ENFORCEMENT.md
├── .rulebook/specs/RULEBOOK.md        — Task management
└── .rulebook/PLANS.md                 — Session scratchpad

Layer 2: Tool-Specific (generated by rulebook update)
├── .claude/rules/*.md                 — Claude Code rules
├── .claude/agents/*.md                — Claude Code specialist agents
├── .cursor/rules/*.mdc               — Cursor contextual rules
├── GEMINI.md                          — Gemini CLI sections
├── .github/copilot-instructions.md   — GitHub Copilot
├── .windsurf/rules/*.md              — Windsurf rules
└── .continue/rules/*.md              — Continue.dev rules

Layer 3: MCP (cross-tool integration)
├── rulebook_session_start/end         — Session management
├── rulebook_rules_list                — Canonical rules
├── rulebook_blockers                  — Task dependency chain
├── rulebook_memory_*                  — Persistent memory
├── rulebook_task_*                    — Task management
└── rulebook_decision/knowledge/learn  — Context intelligence
```

## Key Components

### Rule Engine (`src/core/rule-engine.ts`)

Canonical rules in `.rulebook/rules/` with YAML frontmatter:

```yaml
---
name: no-shortcuts
tier: 1           # 1=prohibition, 2=workflow, 3=standard
alwaysApply: true
filePatterns: ["*"]
tools: ["all"]    # or specific: ["claude-code", "cursor"]
---
```

Projection: `loadCanonicalRules()` → `projectRules()` → 6 formats

### Agent Template Engine (`src/core/agent-template-engine.ts`)

Templates in `templates/agents/<project-type>/` with frontmatter:

```yaml
---
name: shader-engineer
domain: shaders
filePatterns: ["*.hlsl", "*.glsl"]
tier: core        # core → opus, standard → sonnet, research → haiku
checklist:
  - "Which reference source was this based on?"
---
```

Graceful degradation:
- Claude Code → `.claude/agents/*.md` + `.claude/agent-memory/`
- Cursor → `.cursor/rules/agent-*.mdc` (file-glob activated)
- Gemini/Codex → inline "When Editing *.hlsl" sections

### Complexity Detector (`src/core/complexity-detector.ts`)

LOC sampling (500 files max, extrapolated) + language count + structure analysis.

| Tier | LOC | Languages | Rules | Agents |
|------|-----|-----------|-------|--------|
| Small | <10K | 1 | Tier 1 only | None |
| Medium | 10-50K | 1-2 | + Tier 2 | Recommended |
| Large | 50K+ | 2+ | + Tier 2 | Auto-generated |
| Complex | 100K+ | 3+ | + data-flow | + reference workflow |

### Memory Store (`src/memory/memory-store.ts`)

**better-sqlite3** (native) replacing sql.js (WASM):
- WAL journal mode — zero-copy writes
- statSync for size — O(1), no allocation
- <5ms init — no WASM JIT warmup
- Existing `.db` files compatible (standard SQLite3)

### Background Indexer (`src/core/indexer/background-indexer.ts`)

**chokidar** replacing fs.watch:
- OS-level ignore patterns (node_modules filtered before event dispatch)
- Write stabilization (500ms threshold)
- Cross-platform reliability

## Data Flow

```
rulebook init
  ├── detectProject()           — languages, frameworks, tools
  ├── assessComplexity()        — LOC, tier, recommendations
  ├── generateFullAgents()      — AGENTS.md + spec files
  ├── installRule() × N         — Tier 1 + conditional Tier 2
  ├── projectRules()            — canonical → 6 tool formats
  └── generateMultiToolConfigs() — tool-specific directives

rulebook update
  ├── detectProject()
  ├── mergeFullAgents()         — preserves custom blocks
  ├── projectRules()            — re-project canonical rules
  └── generateMultiToolConfigs()
```
