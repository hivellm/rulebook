# Proposal: F2 — First-class `.claude/rules/` generation with `paths:` frontmatter

Source: [docs/analysis/v5.3.0/04-features.md#f2](../../../docs/analysis/v5.3.0/04-features.md)

## Why
All three analyzed projects (tml: 18 rules, project-v: 6, UzEngine: many) hand-maintain `.claude/rules/`. Rulebook only generates AGENTS.md and ignores this directory. Anthropic supports path-scoped rules via `paths:` frontmatter — they only fire when matching files are touched, far more efficient than monolithic AGENTS.md.

## What Changes
- New `templates/rules/` directory shipping initial 8 language templates (TS, JS, Rust, Python, Go, C++, Java, C#).
- Each language template carries YAML frontmatter `paths: [...]`.
- New `src/core/rules-generator.ts` emits `.claude/rules/<lang>.md` for every detected language.
- Each generated file includes a `# Generated by rulebook` sentinel; files without the sentinel are preserved.
- `init` / `update` wire the generator into their flows.
- `rulebook_rules_list` MCP tool extended with source attribution.

## Impact
- Affected specs: `templates/rules/` (new)
- Affected code: new `src/core/rules-generator.ts`, `src/cli/commands.ts`, `src/mcp/rulebook-server.ts`
- Breaking change: NO
- User benefit: language-specific guidance loads only when needed; reduces CLAUDE.md size
