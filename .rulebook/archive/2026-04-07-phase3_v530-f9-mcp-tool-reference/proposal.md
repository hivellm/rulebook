# Proposal: F9 — MCP tool reference auto-generation

Source: [docs/analysis/v5.3.0/04-features.md#f9](../../../docs/analysis/v5.3.0/04-features.md)

## Why
Agents waste cycles spawning sub-agents for tasks a direct MCP tool would handle (tml CLAUDE.md block "Check Skills and MCP Tools BEFORE Spawning Agents" is field evidence). Rulebook should auto-generate a discoverable reference of available MCP tools.

## What Changes
- New `src/core/mcp-reference-generator.ts` walks `.mcp.json` and `.cursor/mcp.json` and emits `.claude/rules/mcp-tool-reference.md` listing discovered tools with one-line summaries.
- Generated file carries the `# Generated by rulebook` sentinel.
- Triggered by `init` and `update`.

## Impact
- Affected specs: `.claude/rules/mcp-tool-reference.md` (generated)
- Affected code: new `src/core/mcp-reference-generator.ts`, `src/cli/commands.ts`
- Breaking change: NO
- User benefit: agents prefer direct MCP calls over expensive sub-agent spawning
